# สูตร: วางแผน / แตกงานเป็นขั้น

1. **เข้าใจเป้าหมาย** — สรุปสิ่งที่ต้องได้ตอนจบ (definition of done) ให้ชัด
2. **สำรวจบริบท** — `list_dir`/`read_file` ดูของที่มีอยู่ กันวางแผนซ้ำหรือชนของเดิม
3. **แตกเป็นงานย่อย** — เรียงเป็นขั้น แต่ละขั้นมี: สิ่งที่ทำ, ผลที่ได้, ใครเหมาะทำ (role), ขึ้นกับขั้นไหน
4. **ชี้ความเสี่ยง** — จุดที่อาจพลาด + วิธีกัน
5. **บันทึก** — `write_file` เป็น `plan/<หัวข้อ>.md` มีหัวข้อ: เป้าหมาย / ขั้นตอน / ความเสี่ยง
6. **รายงาน CEO** — final สรุปแผนสั้น ๆ + ถามถ้ามีจุดต้องตัดสินใจ

กฎ: แผนต้องลงมือทำตามได้จริง (เจาะจง ไม่ลอย) — เลี่ยงคำกว้างเช่น "ปรับปรุงระบบ" ให้บอกว่าปรับอะไร ไฟล์ไหน
