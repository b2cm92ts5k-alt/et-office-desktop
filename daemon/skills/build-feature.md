# สูตร: สร้าง/แก้ฟีเจอร์ในโค้ด

ทำทีละขั้น เน้น "อ่านก่อนเขียน เขียนแล้วเทส":

1. **สำรวจ** — `list_dir` + `read_file` ไฟล์ที่เกี่ยวข้อง เข้าใจโครงสร้าง/สไตล์โค้ดเดิมก่อน
2. **วางแผนสั้น** — บอกว่าจะแตะไฟล์ไหนบ้าง ทำอะไร (1-3 บรรทัด)
3. **เขียน** — `write_file` ทีละไฟล์ ให้เข้ากับสไตล์เดิม (naming/อินเดนต์/คอมเมนต์เท่าที่ไฟล์รอบ ๆ มี) อย่าเขียนทับสิ่งที่ไม่เกี่ยวข้อง
4. **ทดสอบ** — `powershell` รัน build/test/lint เท่าที่โปรเจกต์มี → ถ้า error อ่านแล้วแก้ วนจนผ่าน
5. **ตรวจ** — `git_diff` ดูว่าการเปลี่ยนแปลงตรงกับที่ตั้งใจ ไม่มีของแปลกปลอม
6. **commit (ถ้า CEO อยากให้ commit)** — `git_commit` ข้อความสั้นสื่อความ
7. **รายงาน** — final บอกว่าทำอะไร ไฟล์ไหน เทสผ่านไหม (ถ้าเทสไม่ผ่านให้บอกตามจริง ห้ามอ้างว่าเสร็จทั้งที่ยังพัง)

กฎ: ห้ามบอกว่า "เสร็จแล้ว" ถ้ายังไม่ได้รันเทส/ยังมี error
